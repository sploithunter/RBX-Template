# Authored Map Workflow

This project keeps game logic in Rojo and map geometry in Studio. The handoff is CollectionService tags plus attributes.

## Goal

The first authored map should be a tiny reference map, not a final world:

- `Spawn` floor and `Meadow` floor.
- `Zone` and `AreaZone` markers for `spawn_world`, `spawn_island`, `meadow_island`, `Spawn`, and `Meadow`.
- `SpawnZone` markers for breakables in both areas.
- `EggStand` markers for `basic_egg` and `golden_egg`.
- `TeleportPad` markers between `Spawn` and `Meadow`.
- `Portal` markers between `spawn_island` and `meadow_island`.
- `PODPodium` markers in both areas.

## Create The Reference Map

In Studio edit mode, run:

```lua
-- Paste and run scripts/studio/create_reference_map.luau
```

The script creates `Workspace.AuthoredReferenceMap` plus the necessary `Workspace.Game` marker folders. It refuses to overwrite an existing `AuthoredReferenceMap` unless that folder was generated by the same script.

Save the place after generation if you want to keep the authored markers.

## Verify In Play Mode

Run:

```lua
return require(game:GetService("ReplicatedStorage").Tests.studio.SpawnSafetySmoke).runText({
    zoneId = "Spawn",
    timeoutSeconds = 25,
})
```

Then verify the marker contract:

```lua
return require(game:GetService("ReplicatedStorage").Tests.studio.MapContractSmoke).runText({
    requireAuthored = true,
})
```

Expected result should include authored hooks, for example:

```text
MapContractSmoke passed: authored=17 synthetic=0 ...
```

Then run the behavior tests:

```lua
return require(game:GetService("ReplicatedStorage").Tests.studio.TravelSmoke).runText({
    sourceAreaId = "Spawn",
    targetZoneId = "Meadow",
    timeoutSeconds = 25,
})
```

```lua
return require(game:GetService("ReplicatedStorage").Tests.studio.EggProximitySmoke).runText({
    eggType = "basic_egg",
    timeoutSeconds = 25,
})
```

## Builder Rules

- Visual geometry can move freely.
- Gameplay behavior must move by updating tagged markers, not service code.
- Marker names are for humans; tags and attributes are the contract.
- Keep `configs/areas.lua` and `configs/breakables.lua` as the source of gameplay ids and balance.
- Rojo must not own authored Workspace map folders.

## Promotion Path

1. Start in `map.mode = "auto"` while building.
2. Run `MapContractSmoke` with `requireAuthored = true`.
3. Run travel and egg smoke tests.
4. Temporarily switch to `map.mode = "authored"` before a release candidate to fail loudly if any configured hook is missing.
5. Switch back to `auto` for day-to-day mixed synthetic/authored development.
